# EdgeClaw Box 产品思考

## 一、它是什么

> **EdgeClaw Box 是一台放在桌上的本地 AI 节点。**

它给你局域网内所有设备提供私有 AI 能力——大模型推理、多模态理解、语音交互、语义记忆——数据不出局域网，不按 token 计费，始终开机。

---

## 二、谁买、为什么买

### 目标市场

美国 OPC（一人公司），2980 万个，74% 已在用 AI 工具。

他们不需要被教育"AI 有什么用"，他们已经在用，已经在付钱，已经在痛：

- 月均 API 费 $300-800，随用量线性增长
- 代码和客户数据上传到第三方，隐私风险真实存在
- 云端延迟影响开发体验，想要本地推理

### 但 2980 万这个数字不能直接用

2980 万 OPC 里：

- 大部分只用 ChatGPT Plus（$20/月），不花 $300+/月在 API 上
- 大部分不需要 CUDA
- 大部分不需要 64GB 跑大模型
- 大部分不会为了隐私花 $4000

真正的可触达市场远小于 2980 万。诚实估计，第一年能触达的核心人群可能在万人量级。

这不是坏事——但它意味着必须精准打中对的人，而不是指望大数定律。

### 用户优先级：需要重新想

之前把"独立 AI 开发者"排第一。但有一个问题一直没正视：

**开发者是最在乎推理速度的人。**

Orin 跑 27B 模型的 tok/s 可能只有 5-15（下面第三节详细分析）。云端 API 是 50-100 tok/s。开发者天天用 Cursor 写代码，这个速度差距他们第一分钟就能感受到。

反观合规敏感行业的独立专业人士（律师、医生、会计师）：

- 隐私是刚性需求，不是"有了更好"，是"没有不行"
- 对速度容忍度高（他们用 AI 写摘要、审合同，不需要 100 tok/s）
- 预算更充裕（年收入 $10 万+，$4000 是工具投资）
- 不会和 Mac Mini + Ollama 做性价比对比——他们对比的是"能不能合规用 AI"

**可能应该反过来：合规用户排第一，开发者排第二。**

合规用户的购买理由是二元的：数据能不能不出设备？能。买。

开发者的购买理由是连续谱：tok/s 够不够快？要比一比。值不值？要算一算。

二元决策比连续谱决策的转化率高得多。

## 三、必须正视的硬伤：推理速度

这是之前所有分析里最大的盲区。

### Orin 的内存带宽是短板

LLM 推理（尤其是 decode 阶段）是内存带宽瓶颈任务。tok/s 基本正比于内存带宽。

| 设备 | 内存带宽 | 估算 tok/s（27B Q4） | 价格 |
|---|---|---|---|
| Jetson AGX Orin 64GB | ~200 GB/s | ~8-15 | ~$2000（模组） |
| Mac Studio M4 Ultra 192GB | ~800 GB/s | ~30-50 | ~$4000-5000 |
| RTX 4090 24GB | ~1000 GB/s | 不够跑 27B | ~$1600 |
| 云端 API（GPT-4o） | N/A | 50-100 | 按 token 付费 |

**Orin 跑 27B 的速度可能只有 Mac Studio 的 1/3-1/4，云端 API 的 1/5-1/10。**

这意味着：

- 用 Cursor 写代码，每次补全要等 3-5 秒，而云端 API 是亚秒级
- "省 API 费"的前提是体验可接受，如果体验差太多，省下的钱不如花出去
- 开发者用了一天就会去 r/LocalLLaMA 发帖说"太慢了"

### 这不是致命伤，但必须诚实

不能把 Orin 包装成"和云端 API 一样快的本地替代"。它不是。

它的定位应该是：**为了隐私和自主权，接受合理的速度折损。**

就像 NAS 比 Dropbox 慢，但你选 NAS 是因为数据在自己手里。没人会说 NAS 比 Dropbox 更快——但很多人愿意用 NAS。

**话术不应该是"Stop paying $300/month for API calls"，而是"Your AI, your data, your rules."**

## 四、竞品需要重新想

### Mac Mini M4 不是真正的竞品——Mac Studio M4 Ultra 才是

之前一直拿 Mac Mini M4（$600-800）做对比，得出"我们贵但能力强"的结论。

但同价位的真正竞品是 Mac Studio M4 Ultra：

| 维度 | EdgeClaw Box (Orin 64GB) | Mac Studio M4 Ultra 192GB |
|---|---|---|
| 价格 | ~$3999 | ~$4000-5000 |
| 内存 | 64GB 共享 | 192GB 统一内存 |
| 内存带宽 | ~200 GB/s | ~800 GB/s |
| 能跑的最大模型 | 27B（全精度） | 70B+（全精度） |
| 推理速度 | 慢 | **快 3-4 倍** |
| CUDA | 是 | 否（但 MLX 生态已成熟） |
| 通用计算 | 有限（ARM Linux） | **完整 macOS** |
| 多模态硬件 | 内置摄像头 + 麦克风 + 音响 | **没有** |
| 始终开机设计 | 是 | 一般 |
| 预装 AI 软件栈 | 是 | 否 |

**结论：纯算力和推理速度，Mac Studio 碾压 Orin。**

但 Mac Studio 有几件事做不到：

1. **没有内置感知硬件**——没有摄像头、没有麦克风阵列、没有音响。它是一台电脑，不是一个 AI 设备。
2. **没有预装 AI 软件**——你要自己装 Ollama、配 MiniCPM、写脚本串起来。
3. **没有 CUDA**——虽然 MLX 越来越好，但很多训练/微调工作流仍然依赖 CUDA。
4. **它是一台通用电脑**——你会在上面装其他软件、做其他事，它不是一个专注于 AI 的专用设备。

**所以 EdgeClaw Box 的真正差异化不是"更多 VRAM"或"CUDA"，而是：**

> **它是一个带感知能力的、预配置好的、专用的 AI 设备。Mac Studio 是一台快电脑，EdgeClaw Box 是一个 AI 实体。**

## 五、重新推演：到底卖什么

把前面的分析串起来：

- 纯算力打不过 Mac Studio（速度差 3-4 倍）
- 纯省钱打不过云 API（体验差距太大，很多人宁愿花钱）
- 纯 CUDA 在 2026 年不再是绝对壁垒（MLX 生态已成熟）

那还能卖什么？

### 回到硬件本身看

这台设备有三样东西是 Mac Studio 和云 API 都没有的：

1. **摄像头**
2. **麦克风**
3. **始终开机 + 预装好的 AI 软件**

这三样东西加在一起，创造的不是"更快的本地推理"，而是一种全新的交互形态：

**一个在你工作空间里，始终存在、能看能听能说、有记忆的 AI。**

这才是真正没有竞品的东西。

Mac Studio 很快，但它没有眼睛和耳朵。
Echo 能听，但它没有大脑。
手机 AI 能看能听，但你得掏出来、打开 App、对着它说。

EdgeClaw Box 是第一个把"足够强的大脑 + 持续感知 + 始终在线"做到一起的消费级设备。

### 但这和前面"不要做太多"的结论矛盾吗？

不矛盾。关键在于讲故事的顺序和每一层的完成度要求。

之前的逻辑是：v1 先卖"本地 AI 节点"（省 API 费），感知是后面的事。

但重新推演之后，这个顺序可能有问题：

- "本地 AI 节点"这个卖点，Orin 的推理速度扛不住和 Mac Studio 的正面比较
- "省 API 费"这个 ROI 故事，在体验差距面前没那么有说服力
- 而"感知 + 始终在线"恰恰是你有而所有竞品都没有的

**也许应该反过来：v1 的核心卖点不是"本地 AI 节点"，而是"你桌上的 AI"。**

| 定位 | 核心竞争对手 | 你赢在哪 |
|---|---|---|
| 本地 AI 节点 | Mac Studio + Ollama | 你输（速度慢、内存小） |
| 桌上的 AI（能看能听有记忆） | Echo / Siri / 手机 AI | 你赢（真正的大脑 + 记忆） |

**选一个你能赢的战场打。**

## 六、修正后的产品定义

### 一句话

> **EdgeClaw Box 是一个始终在场的 AI——它在你桌上，能听、能看、能说、有记忆，数据永远不出你的空间。**

不叫它"服务器"，不叫它"节点"。叫它"你桌上的 AI"。

### 三层卖点

```
第一层（v1 核心，决定生死）：桌上的 AI

  它在你桌上，你开口就能和它说话
  它能看到你面前的东西
  它的回答基于真正的大模型推理，不是 Siri 级别的
  数据完全本地，你有绝对控制权
  
  验证标准：第一次和它对话，觉得"这比 Siri/Echo 好太多了"
  竞争对手：Echo / Siri / Google Home → 你赢

第二层（v1 有，是留存理由）：记忆

  说"记住这件事"→ 它存下来
  过两周问"之前说过什么"→ 它找出来
  用得越久，记忆越深，越不想换
  
  验证标准：用了 1 个月后，记忆里有了不想丢的东西
  
第三层（v1 提供，差异化来源）：本地 AI 能力输出

  它同时也是一台 AI 服务器
  你的电脑、手机可以通过 API 调用它的模型能力
  省 API 费、保隐私，但速度不会比云端快——这是诚实的权衡

  验证标准：Cursor 指向它能正常写代码（能用，虽然不如云端快）

第四层（中期实验）：被动感知

  持续监听 + 自动提取有价值信息
  如果做成：全新品类
  如果没做成：不拖死主产品
```

### 和之前版本的核心区别

之前：先卖"本地 AI 节点"（省 API 费），感知是后面的事。

现在：先卖"桌上的 AI"（交互体验），API 能力是附带的。

为什么变了：因为在"本地 AI 节点"这个赛道上，Orin 的推理速度打不过 Mac Studio。但在"桌上的 AI"这个赛道上，没有人做到过。

## 七、凭什么卖 $3999

不再和 Mac Studio 比算力（会输），不再和云 API 比速度（会输）。

和以下东西比：

| 对比对象 | 它能做的 | EdgeClaw Box 多了什么 |
|---|---|---|
| Echo / HomePod ($100-300) | 听指令、放音乐、智能家居 | 真正的大模型推理 + 视觉 + 记忆 |
| ChatGPT Plus ($20/月) | 强大 AI，但要打开 App 才能用 | 始终在场、语音交互、不用掏手机 |
| Rabbit R1 / Humane AI Pin ($200-700) | 可穿戴 AI | 算力远超、不需充电、固定在工作场景 |
| 云 API ($300+/月) | 强大但贵，隐私不可控 | 一次买断、数据本地、18 个月回本 |

$3999 的定价锚点不应该是硬件成本，而是：

- 对合规用户：这是**唯一**让你合法使用 AI 处理客户数据的方案。合规的价值无法用 $3999 衡量。
- 对重度 AI 用户：月均 $300 API 费 × 18 个月 = $5400。$3999 一次买断，之后零成本。
- 对所有人：一台真正的 AI 设备，不是一个 App，不是一个订阅，是你拥有的、在你空间里的 AI。

## 八、核心风险

### 1. 推理速度让开发者失望

Orin 跑 27B 的 tok/s 约 8-15，云端 API 50-100。开发者会第一时间感受到差距。

**对策：不把开发者的代码补全体验作为 v1 核心卖点。诚实标注性能参数。可以推荐用更小的模型（14B/9B）来换速度。** 同时，验证实际 tok/s——如果实测比估算好，这个风险会降低。

### 2. "桌上的 AI"这个品类还没有被验证

Echo 验证了"语音助手"有市场，但"有大脑、有记忆的桌上 AI"是一个全新品类。用户到底愿不愿意和一台桌上设备持续互动？不知道。

**对策：自己先用 3 个月。如果自己都不每天用，别人更不会。**

### 3. 软件体验跟不上硬件

硬件用户容忍度低。$3999 买了一台设备，语音卡顿、记忆找不回来、视觉识别不准——任何一个都会导致退货。

**对策：v1 的功能范围要极小，但每个功能必须稳定。宁可只有语音对话 + 记忆两个功能，也不要做 10 个半成品。**

### 4. 被动感知的隐私顾虑

一台始终开着摄像头和麦克风的设备，很多人会不安。

**对策：物理开关（硬件级关闭摄像头/麦克风），默认关闭被动感知，用户主动开启。**

## 九、接下来做什么

### 第一件事：跑真实的 tok/s（本周）

在 Orin 上实测 Qwen3.5 27B / 14B / 9B 的推理速度。

这个数字决定一切：

- 如果 27B 能到 15+ tok/s → 可以作为主力模型
- 如果 27B 只有 5-8 tok/s → 主力应该换成 14B，27B 作为高精度备选
- 不管结果如何，把数字写出来，不要藏

### 第二件事：让语音对话体验达标（2 周）

全双工语音对话是"桌上的 AI"这个定位的生死线。

测试标准：
- 说完话到开始回复 < 2 秒
- 说话过程中可以打断
- 连续对话 30 分钟不崩溃

### 第三件事：让记忆可用（2 周）

- 实现语义检索（本地 embedding + 向量匹配）
- 测试：存 50 条记忆，用自然语言问，能找回 80%+

### 第四件事：自己每天用（4 周）

把设备放在自己桌上，每天用。记录：
- 每天用了几次
- 哪些场景真的有用
- 哪些场景想用但体验不行
- 有没有一天忘了用

如果自己用不起来，停下来想为什么，而不是继续往前冲。

### 第五件事：拍视频（1 周）

只有在自己每天用、确认体验值得之后，才拍。

## 十、最终判断

### 方向需要修正

之前的定位"本地 AI 节点，省 API 费"有一个没正视的问题：Orin 的推理速度在同价位硬件中不占优势。用这个定位去打，会被 Mac Studio + Ollama 正面碾压。

修正后的定位：**一个在你桌上、能看能听有记忆的 AI。** 这个赛道上没有竞品。

### 用户优先级需要修正

开发者对速度最敏感，是最容易因为 tok/s 不够而失望的人群。合规敏感行业的专业人士对隐私是刚性需求，对速度容忍度高，购买决策更简单。

**合规用户排第一，开发者排第二。**

### 必须验证的一个数字

Orin 上 Qwen3.5 27B 的实际 tok/s。

如果 < 10 tok/s，整个"本地推理替代云 API"的故事需要大幅调整，产品应该主打交互体验和隐私，而不是算力。

如果 > 15 tok/s，算力故事可以作为第二卖点保留。

**在这个数字跑出来之前，所有关于性能的承诺都是猜测。**

### 一句话

> **先做一个在你桌上的、能听能看有记忆的 AI，让它成为你离不开的存在。算力是它的基础设施，不是它的卖点。**
